# Usability Testing Script - CleanSpace App

## Pre-Test Instructions

**Moderator**: "Thank you for participating in this usability test. Today we'll be testing the CleanSpace mobile app, which is a platform connecting clients with cleaning service providers. This session will take approximately 45 minutes.

Please remember:
- There are no right or wrong answers
- We're testing the app, not you
- Please think aloud as you use the app
- Feel free to ask questions if you get stuck
- I may ask follow-up questions during the session

Do you have any questions before we begin?"

---

## Test Session Structure

### Phase 1: Introduction and Background (5 minutes)
**Questions**:
1. How familiar are you with mobile apps?
2. Have you used any service marketplace apps before? (Uber, Airbnb, etc.)
3. What device are you most comfortable using? (iOS/Android)

### Phase 2: Task-Based Testing (35 minutes)

#### Task 1: Account Creation (5 minutes)
**Instructions**: 
"Please create a new account as a Client user. Use any information you'd like - this is a test environment."

**Observations to note**:
- Time to complete
- Any confusion points
- Error messages encountered
- Overall ease of process

**Follow-up Questions**:
- How clear were the registration instructions?
- Was anything confusing about this process?

---

#### Task 2: Post a Job Listing (7 minutes)
**Instructions**:
"Now that you're logged in as a Client, please create a new cleaning job posting. You can use any details you'd like."

**Observations to note**:
- Can user find the "Post Job" option?
- Time to complete all fields
- Any missing or unclear fields?
- Success confirmation visibility

**Follow-up Questions**:
- How easy was it to find where to post a job?
- Were all the required fields clear?
- How confident are you that your job was posted successfully?

---

#### Task 3: Browse and Filter Jobs (6 minutes)
**Instructions**:
"Please browse the available job listings. Try to find a job in a specific location or price range if possible."

**Observations to note**:
- How user navigates job listings
- Filter usage (if attempted)
- Time to find relevant jobs
- Job detail view usability

**Follow-up Questions**:
- How easy was it to browse through jobs?
- Did you notice the filter options? Were they clear?
- What information is most important to you when viewing jobs?

---

#### Task 4: Apply to a Job (5 minutes)
**Instructions**:
"Now imagine you're a cleaner looking for work. Please apply to one of the available jobs."

**Observations to note**:
- Can user switch context/role? (or need separate account)
- Application process clarity
- Confirmation of application
- Any errors during process

**Follow-up Questions**:
- Was the application process straightforward?
- How do you know if your application was successful?
- What would make this process better?

---

#### Task 5: Manage Notifications (6 minutes)
**Instructions**:
"Please open your notifications inbox. See if there are any new notifications, and try marking some as read."

**Observations to note**:
- Time to find notifications
- Clarity of notification content
- Ease of marking as read
- Badge count accuracy

**Follow-up Questions**:
- Are the notifications clear and informative?
- Is it easy to see which notifications are new?
- How useful are the notifications?

---

#### Task 6: Edit Profile (6 minutes)
**Instructions**:
"Please update your profile information. Try changing your name, adding a bio, and uploading a profile picture."

**Observations to note**:
- Time to find profile settings
- Ease of editing fields
- Photo upload success/failure
- Save confirmation

**Follow-up Questions**:
- How easy was it to update your profile?
- Were there any issues with uploading a photo?
- Is there anything you'd like to change about your profile that you couldn't?

---

### Phase 3: Overall Feedback (5 minutes)

**Final Questions**:
1. On a scale of 1-10, how would you rate the overall user experience? Why?
2. What did you like most about the app?
3. What frustrated you the most?
4. If you could change one thing about the app, what would it be?
5. Would you recommend this app to a friend? Why or why not?
6. Any other comments or suggestions?

---

## Post-Test
**Moderator**: "Thank you so much for your time and feedback. Your insights are valuable for improving the app. Do you have any final thoughts or questions?"

---

## Notes for Moderator
- Allow participants to struggle for 1-2 minutes before offering hints
- Note body language and facial expressions indicating frustration
- Take screenshots or notes of specific issues encountered
- Record task completion times accurately
- Be neutral and avoid leading participants
- Thank participants and explain how feedback will be used

---

## Test Environment Checklist
- [ ] App installed and running on test device
- [ ] Test accounts prepared (Client, Cleaner, Agency)
- [ ] Test data available (sample jobs, notifications)
- [ ] Screen recording software ready (optional)
- [ ] Timer ready for tracking task completion
- [ ] Note-taking materials ready
- [ ] Quiet environment for testing
